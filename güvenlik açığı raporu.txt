1. Giriş
Amaç: Sistem üzerinde yapılan güvenlik incelemeleri sonucunda, bir Nmap taraması gerçekleştirilmiş ve aşağıdaki açık portlar tespit edilmiştir:

yaml
Kopyala
Düzenle
21, 22, 25, 53, 80, 110, 111, 143, 443, 465, 587, 993, 995, 3211, 3306
Bu portların detaylı incelenmesi sonucunda özellikle SSH servisinde kritik güvenlik riskleri saptanmıştır.

2. Bulgular
🔴 Kritik Seviyede Açıklar
1. OpenSSH 8.2p1 (22/tcp)

Risk: Uzaktan Kod Çalıştırma (CVE-2023-38408, CVE-2020-15778)

Etkisi: Yetkisiz saldırganın sistem üzerinde shell elde etmesi.

Sızma Noktası: SSH servisi → sistem yönetim yetkileri.

Öneri:

OpenSSH güncel sürüme yükseltilmeli

brute-force koruması (ör. Fail2Ban) aktif edilmeli

yalnızca gerekli IP’lere erişim izni verilmelidir.

2. Apache HTTPD (80/tcp)

Risk: Slowloris DoS (CVE-2007-6750)

Etkisi: Web sitesinin hizmet dışı kalması.

Sızma Noktası: HTTP üzerinden çoklu bağlantı açılarak sunucu kaynaklarının tükenmesi.

Öneri:

mod_reqtimeout veya reverse proxy kullanılması

Web Application Firewall (WAF) eklenmesi.

🟠 Orta Seviyede Açıklar
3. Exim SMTP (587/tcp)

Risk: Geçmişte ciddi RCE zafiyetleri (örn. CVE-2019-10149).

Etkisi: Yetkisiz mail gönderimi veya sistem manipülasyonu.

Sızma Noktası: SMTP doğrulama / relay yapılandırması.

Öneri:

Exim güncel sürüme yükseltilmeli

relay kısıtlamaları uygulanmalıdır.

4. RPCbind (111/tcp)

Risk: RPC servisleri üzerinden bilgi sızması / yetki yükseltme.

Etkisi: Dosya sistemine yetkisiz erişim veya root yetkisi elde edilmesi.

Sızma Noktası: rpcbind üzerinden sunucu iç servislerine erişim.

Öneri:

Kullanılmıyorsa tamamen kapatılmalı

gerekiyorsa sadece localhost erişimine izin verilmelidir.

3. Genel Risk Senaryosu
Muhtemel saldırı zinciri şu şekildedir:

SSH servisi üzerinden sunucuya sızma.

FTP veya web dizinine erişim sağlama.

public_html dizinine spam HTML dosyası eklenmesi.

Google Search Console tarafından sitenin spam olarak işaretlenmesi.

➡️ Bu senaryo, incelenen saldırı izleri ile uyumludur.

4. Önerilen Önlemler
OpenSSH, Exim ve Apache gibi kritik servislerin güncellenmesi.

Kullanılmayan servislerin (örn. rpcbind) kapatılması.

Güçlü parola politikası + brute-force engelleyici çözümler.

Web uygulaması için dosya bütünlüğü kontrolü ve log izleme.

ClamAV veya benzeri bir antivirüs taraması ile saldırganın bırakmış olabileceği arka kapıların temizlenmesi.

Sunucu loglarının düzenli olarak analiz edilmesi.

5. Sonuç
Sistemde özellikle SSH servisi üzerinden kritik riskler barındırdığı tespit edilmiştir.
Bu açıkların kapatılmaması durumunda, saldırganların web sitesine yeniden erişip spam içerik eklemesi yüksek ihtimaldir.

Benim erişim yetkilerim sınırlı olduğundan doğrudan malware temizliği yapılamamıştır.
Bu sebeple Blue Team ekibinin tam güncelleme ve güvenlik taraması yapması önerilir.
